# Plan: Early Report Scaffolding for Sprint Runner

## Context

When sprint phases hit the `max_turns` limit (50 turns), the Claude agent never writes the `phase-N-result.md` file because report writing is the last action in the Completion Protocol. This caused 3 of 5 phases in a recent sprint to finish as `pass_no_report` тАФ the sprint continued but no structured per-task report existed.

The fix: create a scaffold result file BEFORE the agent starts, and modify the prompt so the agent updates it incrementally rather than writing a monolithic report at the end. Always-on, no opt-in flag.

## Changes

### 1. Add `_parse_phase_tasks()` to `executor.py`

**File**: `src/superclaude/cli/sprint/executor.py`
**Location**: New function at module level, before `execute_sprint()`

Parses a phase tasklist file to extract task metadata:
- Task ID + title from headings: `^###\s+(T\d{2}\.\d{2})\s+--\s+(.+)$`
- Tier from metadata table: `^\|\s*Tier\s*\|\s*(STRICT|STANDARD|LIGHT|EXEMPT)\s*\|`
- Returns `list[dict]` with keys `id`, `title`, `tier` (defaults to `"UNKNOWN"`)
- Gracefully returns `[]` on `OSError` or no matches

### 2. Add `_scaffold_result_file()` to `executor.py`

**File**: `src/superclaude/cli/sprint/executor.py`
**Location**: New function after `_parse_phase_tasks()`

Creates scaffold at `config.result_file(phase)`:
- YAML frontmatter: `phase`, `tasks_total`, `tasks_passed: 0`, `tasks_failed: 0`, `scaffolded: true`
- Deliberately **no** `status:` field, **no** `EXIT_RECOMMENDATION:` token
- Per-task markdown table with all rows as `pending` / `---`
- Empty "Files Modified" and "Blockers" sections
- Ensures `results_dir` exists via `mkdir(parents=True, exist_ok=True)`
- Wrapped in try/except тАФ logs warning on failure, does not abort sprint

### 3. Insert scaffold call in `execute_sprint()` loop

**File**: `src/superclaude/cli/sprint/executor.py`
**Location**: Between `proc_manager = ClaudeProcess(config, phase)` (line ~83) and `proc_manager.start()` (line ~84)

```python
proc_manager = ClaudeProcess(config, phase)
_scaffold_result_file(config, phase)   # NEW
proc_manager.start()
```

### 4. Modify `build_prompt()` in `process.py`

**File**: `src/superclaude/cli/sprint/process.py`
**Location**: Replace the "Completion Protocol" section (lines ~69-82) in `build_prompt()`

Replace with "Reporting Protocol" that tells the agent:
1. A scaffold report already exists at `{result_file}` with all tasks as `pending`
2. After each task: update that task's row from `pending` to `pass`/`fail`, add evidence, increment frontmatter counters
3. When all tasks done: add `status: PASS|FAIL|PARTIAL` to frontmatter and `EXIT_RECOMMENDATION: CONTINUE|HALT` at the end

## Status Classification Impact (no changes to `_determine_phase_status`)

| Scenario | Before | After |
|----------|--------|-------|
| Agent completes + EXIT_RECOMMENDATION: CONTINUE | PASS | PASS |
| Agent partial + max_turns | PASS_NO_REPORT | PASS_NO_SIGNAL (scaffold w/ partial data) |
| Agent doesn't touch scaffold | PASS_NO_REPORT | PASS_NO_SIGNAL (scaffold, all pending) |
| Agent writes HALT | HALT | HALT |

Both `PASS_NO_REPORT` and `PASS_NO_SIGNAL` are `is_success=True` тАФ sprint continues in all cases. The improvement: cases 2 & 3 now have a structured file with task listings.
