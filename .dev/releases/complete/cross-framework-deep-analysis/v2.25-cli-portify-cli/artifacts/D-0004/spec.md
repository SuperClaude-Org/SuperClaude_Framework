## 1. Architecture Decision Notes

### Core Architectural Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Execution model | Sequential-only | Simplifies executor; all 12 steps have linear dependency chain |
| Domain model inheritance | Extend pipeline base types | Consistent API; tight framework coupling; re-uses gate/step infrastructure |
| Gate enforcement | All 12 gates BLOCKING | Every step output feeds downstream; no independent quality-only checks |
| Turn budget | TurnLedger from sprint.models | Established pattern; not in pipeline.models |
| User review gates | Pipeline pause + approval YAML | Enables async review; user inspects artifacts before resume |
| Convergence loop | Step-internal state machine | Max 3 iterations; CONVERGED or ESCALATED terminal conditions |
| Prompt splitting | Executor responsibility (>300 lines) | Content/routing separation; executor has aggregate size visibility |
| Module overwrite | Marker-based (Generated by / Portified from) | Safe re-run; prevents accidental overwrites of non-portified modules |
| Kill signal | SIGTERM ‚Üí 10s grace ‚Üí SIGKILL | Inherit ClaudeProcess.terminate() pattern from pipeline.process |
| Debug logging | NullHandler vs FileHandler (debug.log) | Zero overhead by default; structured events on --debug |

### Verified Import Paths

## 2. Finalized Interface Assumptions

### `PortifyConfig(PipelineConfig)` Interface Contract

Domain model inherits from `PipelineConfig`. All `PipelineConfig` fields are available:

| Field | Type | Default | Purpose |
|-------|------|---------|---------|
| `work_dir` | `Path` | `Path(".")` | Working directory for artifacts |
| `dry_run` | `bool` | `False` | Limit to phases 0-2 |
| `max_turns` | `int` | `100` | Base field; cli-portify default is 200 |
| `model` | `str` | `""` | Claude model identifier |
| `permission_flag` | `str` | `"--dangerously-skip-permissions"` | Subprocess permission flag |
| `debug` | `bool` | `False` | Enable debug.log |
| `grace_period` | `int` | `0` | Unused for cli-portify (all gates BLOCKING) |

`PortifyConfig` adds: `workflow_path`, `cli_name`, `output_dir`, `stall_timeout`, `stall_action`, `resume_from`.

### `PortifyStep(Step)` Interface Contract

Inherits `Step` fields. Adds:

| Field | Type | Default |
|-------|------|---------|
| `phase_type` | `PortifyPhaseType` | `PREREQUISITES` |
| `is_programmatic` | `bool` | `False` |
| `convergence_max_iterations` | `int` | `0` (3 for step 11) |

### `PortifyStepResult(StepResult)` Interface Contract

Inherits `StepResult` fields. Overrides `status` type to `PortifyStatus`. Adds:

| Field | Type | Default |
|-------|------|---------|
| `status` | `PortifyStatus` | `PENDING` |
| `exit_code` | `int \| None` | `None` |
| `output_bytes` | `int` | `0` |
| `error_bytes` | `int` | `0` |
| `artifacts_produced` | `list[str]` | `[]` |
| `gate_details` | `dict` | `{}` |
| `convergence_iterations` | `int` | `0` |
| `quality_scores` | `dict[str, float]` | `{}` |

### Gate System Interface

`GateCriteria` is stable. Gate semantic check functions return `bool` (per `SemanticCheck.check_fn`), but `gate_passed()` in `pipeline.gates` returns `tuple[bool, str]` (passed, reason). All 12 gate definitions use:
- `required_frontmatter_fields: list[str]`
- `min_lines: int`
- `enforcement_tier: "STRICT" | "STANDARD"`
- `semantic_checks: list[SemanticCheck]`

### Status Classification Interface

```python
# PortifyStatus.PASS_NO_SIGNAL ‚Üí triggers retry (retry_limit >= 1)
# PortifyStatus.PASS_NO_REPORT ‚Üí does NOT trigger retry (passing outcome)
# exit_code 124 ‚Üí PortifyStatus.TIMEOUT
# other non-zero exit_code ‚Üí PortifyStatus.ERROR
```
