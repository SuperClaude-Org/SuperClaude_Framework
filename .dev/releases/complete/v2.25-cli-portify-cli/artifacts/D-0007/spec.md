## Collision Detection Algorithm

1. Normalize derived CLI name to snake_case (replace `-` with `_`).
2. Check if the name (kebab or snake) is in `_KNOWN_CLI_MODULES` — built-in reserved names.
   - If yes, raise `NAME_COLLISION` (no overwrite allowed for core modules).
3. Locate `cli_root` = `src/superclaude/cli/` relative to project root.
4. Check if `cli_root/<snake_name>/` exists on the filesystem.
5. If it exists:
   - Read `__init__.py` and look for markers: `Generated by` or `Portified from`.
   - If marker present: allow overwrite (return no errors).
   - If marker absent: return `NAME_COLLISION` error.
6. If no collision: return empty error list.

## Output Validation Algorithm

1. If `output_dir` is set:
   - If directory exists but is not writable (per `os.access(path, os.W_OK)`), raise `OUTPUT_NOT_WRITABLE`.
   - If directory does not exist, attempt `mkdir(parents=True)`. On `PermissionError`/`OSError`, raise `OUTPUT_NOT_WRITABLE`.

## Error Codes

| Code | Condition |
|------|-----------|
| `NAME_COLLISION` | CLI name matches existing non-portified module |
| `OUTPUT_NOT_WRITABLE` | Output path not writable or cannot be created |
