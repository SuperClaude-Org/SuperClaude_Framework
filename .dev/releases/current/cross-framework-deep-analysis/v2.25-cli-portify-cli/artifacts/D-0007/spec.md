## Collision Detection Algorithm

Implemented in `src/superclaude/cli/cli_portify/config.py::_check_collision()`.

### Rules

1. **Reserved name check**: If derived CLI name (or its snake_case form) matches any known CLI module (`sprint`, `roadmap`, `tasklist`, `cleanup-audit`, `audit`, `pipeline`, `main`) → `NAME_COLLISION`.

2. **Filesystem check**: Locate `src/superclaude/cli/` root via `_find_cli_root()`. If `<root>/<cli_name_snake>/` exists:
   - Check `__init__.py` for portification markers (`Generated by` or `Portified from`)
   - If markers found → **allow overwrite** (no error)
   - If markers absent → `NAME_COLLISION`

3. **No collision** → return empty list.

### Output Writability Validation

Implemented in `validate_portify_config()`:

1. If `output_dir` exists and is not writable → `OUTPUT_NOT_WRITABLE`
2. If `output_dir` doesn't exist → attempt `mkdir(parents=True)`:
   - Success → OK
   - `PermissionError` / `OSError` → `OUTPUT_NOT_WRITABLE`

### Error Codes

| Code | Trigger |
|------|---------|
| `NAME_COLLISION` | Derived name matches existing non-portified module |
| `OUTPUT_NOT_WRITABLE` | Output path cannot be created or written to |

def is_portified_module(init_path: Path) -> bool:
