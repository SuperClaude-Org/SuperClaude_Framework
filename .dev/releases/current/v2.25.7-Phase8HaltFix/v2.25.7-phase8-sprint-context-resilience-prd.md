# PRD: v2.25.7 Phase 8 — Sprint Context Resilience

**Version**: 2.0
**Date**: 2026-03-16
**Status**: Final
**Release**: v2.25.7
**Branch**: `v2.25.7-phase8`
**Compliance Tier**: STRICT (multi-file orchestration subsystem)

---

## 1. Problem Statement

The `superclaude sprint run` executor launches a single Claude subprocess per phase. When that subprocess exhausts its context window (200K tokens for Sonnet), the Anthropic API returns `"Prompt is too long"` with exit code 1. The executor's current `detect_prompt_too_long()` function scans only the NDJSON stdout file (`output_path`), missing cases where the error surfaces on stderr. Additionally, `_determine_phase_status()` does not pass the error file to detection, leaving a defense-in-depth gap. The phase-specific directory isolation system (`setup_isolation()`) exists but is never wired into the execution loop, so the full `tasklist-index.md` (~14K tokens) remains accessible to subprocess agents via `@` file resolution, accelerating context exhaustion. The `build_prompt()` function lacks a sprint-context summary header that would orient the agent without requiring it to discover project structure independently. Finally, `SprintLogger.write_phase_result()` does not route `PASS_RECOVERED` phases to screen output, and `FailureClassifier.classify()` constructs its `output_file` path via a brittle hardcoded expression instead of using `config.output_file(phase)`. These gaps were exposed during the CLI Portify sprint, where Phase 2 (7 tasks, 192 tests passing) crashed at turn 106 while writing the final completion report, halting the sprint at a cost of $24.96 despite all implementation work being complete.

## 2. Solution Architecture

| # | Change | File(s) | Effort | What It Prevents |
|---|--------|---------|--------|------------------|
| S3-A | Wire `setup_isolation()` into `execute_sprint()` — per-phase directory creation, scoped working dir, cleanup | `executor.py` | S | Subprocess loading `tasklist-index.md` (~14K tokens wasted) |
| S3-B | Add `env_vars` support to `ClaudeProcess.__init__` and pipeline `build_env()` | `process.py`, `pipeline/process.py` | S | Isolation directory not propagated to subprocess environment |
| S3-C | Startup cleanup of orphaned `.isolation/` directories | `executor.py` | XS | Stale isolation dirs accumulating from crashed runs |
| S3-D | Add `## Sprint Context` summary header to `build_prompt()` | `process.py` | XS | Agent wastes tokens discovering project structure |
| S2-D | Extend `detect_prompt_too_long()` to also scan `error_path` | `monitor.py` | XS | Context exhaustion error on stderr goes undetected |
| S2-E | Pass `error_file` from `_determine_phase_status()` to detection | `executor.py` | XS | Defense-in-depth gap — stderr never checked |
| FIX-1 | Route `PASS_RECOVERED` to screen in `write_phase_result()` | `logging_.py` | XS | Recovered phases produce no screen output |
| FIX-2 | Replace hardcoded `output_file` path in `FailureClassifier.classify()` | `diagnostics.py` | XS | Brittle path breaks if directory layout changes |

## 3. Functional Requirements

### S3-A: Isolation Wiring (`executor.py`)

| ID | Requirement | Priority |
|----|-------------|----------|
| S3-R01 | Before spawning each phase subprocess in `execute_sprint()`, create a per-phase isolation directory at `config.results_dir / ".isolation" / f"phase-{phase.number}"` and copy only the phase file into it using `shutil.copy2()` | P0 |
| S3-R02 | Pass the per-phase isolation directory as `scoped_work_dir` to `ClaudeProcess` so the subprocess cannot resolve `tasklist-index.md` or other phase files via `@` | P0 |
| S3-R03 | In the `finally` block of the phase execution loop, remove the per-phase isolation directory using `shutil.rmtree(phase_isolation_dir, ignore_errors=True)` | P0 |

### S3-B: ClaudeProcess `env_vars` Support (`process.py` + `pipeline/process.py`)

| ID | Requirement | Priority |
|----|-------------|----------|
| S3-R07 | Add keyword-only parameter `env_vars: dict[str, str] | None = None` to `ClaudeProcess.__init__()` in `src/superclaude/cli/sprint/process.py`. Store as `self._extra_env_vars`. Pass through to pipeline base class or merge into the environment at subprocess spawn | P0 |
| S3-R08 | Add keyword-only parameter `env_vars: dict[str, str] | None = None` to `build_env()` in `src/superclaude/cli/pipeline/process.py`. When not None, merge the provided dict into the returned environment (after the `os.environ.copy()` call, before return) | P0 |

### S3-C: Startup Orphan Cleanup (`executor.py`)

| ID | Requirement | Priority |
|----|-------------|----------|
| S3-R04 | At the start of `execute_sprint()`, before the phase loop begins, check for and remove any existing `.isolation/` directory under `config.results_dir` using `shutil.rmtree(isolation_base, ignore_errors=True)` where `isolation_base = config.results_dir / ".isolation"` | P1 |

### S3-D: `build_prompt()` Summary Header (`process.py`)

| ID | Requirement | Priority |
|----|-------------|----------|
| S3-R05 | Add a `## Sprint Context` section to `build_prompt()` output containing: sprint name (from `config.release_name` or release dir name), phase number and total phases, artifact root path (`config.release_dir`), results directory (`config.results_dir`), and prior-phase artifact directories (list of `artifacts/phase-{N}/` for all N < current phase) | P0 |
| S3-R06 | Add the instruction `"All task details are in the phase file. Do not seek additional index files."` within the `## Sprint Context` section | P1 |

### S2-D: `detect_prompt_too_long()` Error Path Extension (`monitor.py`)

| ID | Requirement | Priority |
|----|-------------|----------|
| S2-R08 | Add keyword-only parameter `error_path: Path | None = None` to `detect_prompt_too_long()`. When `error_path` is not None, scan it for the `PROMPT_TOO_LONG_PATTERN` using the same last-10-lines logic applied to `output_path`. Return `True` if the pattern is found in either file | P1 |

### S2-E: `_determine_phase_status()` Error File Plumbing (`executor.py`)

| ID | Requirement | Priority |
|----|-------------|----------|
| S2-R08b | Add keyword-only parameter `error_file: Path | None = None` to `_determine_phase_status()` with default `None` | P1 |

| S2-R08d | Inside `_determine_phase_status()`, pass `error_path=error_file` to `detect_prompt_too_long()` when `error_file` is not None | P1 |

## 5. Test Requirements

All new tests go in `tests/sprint/test_phase8_halt_fix.py` (extending the existing 12-test file). Tests are grouped into three classes.

### Class: `TestIsolationLifecycle` (S3-A, S3-C)

| Test ID | Test Name | Acceptance Criteria |
|---------|-----------|---------------------|
| T04.01 | `test_isolation_dir_created_before_subprocess` | Given a phase, when `execute_sprint()` reaches the subprocess launch point, then `config.results_dir / ".isolation" / f"phase-{N}"` exists and contains exactly one file matching `phase.file.name` |
| T04.02 | `test_isolation_dir_cleaned_after_phase_completes` | Given a phase that completes with `exit_code=0`, when the phase loop iteration ends, then the per-phase isolation directory no longer exists |
| T04.03 | `test_isolation_dir_cleaned_after_phase_fails` | Given a phase that fails with `exit_code=1`, when the finally block executes, then the per-phase isolation directory no longer exists |
| T04.04 | `test_orphaned_isolation_dirs_cleaned_on_startup` | Given a pre-existing `.isolation/` directory under `config.results_dir`, when `execute_sprint()` begins (before the phase loop), then the orphaned `.isolation/` directory is removed |

### Class: `TestPromptAndDetection` (S3-D, S2-D)

| Test ID | Test Name | Acceptance Criteria |
|---------|-----------|---------------------|
| T04.05 | `test_summary_header_in_prompt` | Given a `ClaudeProcess` constructed with valid config/phase, when `build_prompt()` is called, then the returned string contains `## Sprint Context`, the phase number, and the string `"Do not seek additional index files"` |
| T04.06 | `test_detect_prompt_too_long_scans_error_file` | Given an error file containing `"Prompt is too long"` in the last 10 lines and a clean output file, when `detect_prompt_too_long(output_path, error_path=error_path)` is called, then it returns `True` |
| T04.07 | `test_detect_prompt_too_long_error_path_none` | Given `error_path=None`, when `detect_prompt_too_long(output_path)` is called, then behavior is identical to the current implementation (no error file scanned, no exception) |

### Class: `TestFixesAndDiagnostics` (FIX-1, FIX-2, S2-E)

| Test ID | Test Name | Acceptance Criteria |
|---------|-----------|---------------------|
| T04.08 | `test_logger_pass_recovered_screen_output` | Given a `PhaseResult` with `status=PhaseStatus.PASS_RECOVERED`, when `SprintLogger.write_phase_result()` is called, then `_screen_info()` is invoked (not silently dropped) |
| T04.09 | `test_classifier_uses_config_output_file` | Given a `DiagnosticBundle` with a valid config, when `FailureClassifier.classify()` resolves the output file path, then it uses `config.output_file(phase)` (not the hardcoded `parent.parent / "results"` expression). Verified by mocking `config.output_file()` and asserting it is called |
