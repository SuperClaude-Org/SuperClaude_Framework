## 1. Problem Statement

The roadmap pipeline crashes with `OSError: [Errno 7] Argument list too long` at the
`spec-fidelity` step when combined input files exceed 128 KB. The pipeline's embed guard
(`_EMBED_SIZE_LIMIT = 200 KB`) is set above the Linux kernel's hard per-argument limit
(`MAX_ARG_STRLEN = 128 KB`), creating a 72 KB dead zone where content passes the guard
but crashes `subprocess.Popen`.

### 1.1 Evidence

| Evidence | Source | Impact |
|----------|--------|--------|
| `OSError: [Errno 7]` at spec-fidelity step | Pipeline run against `v2.25-spec-merged.md` | Pipeline halted, no recovery |
| `_EMBED_SIZE_LIMIT = 200 * 1024  # 100 KB` — comment/value mismatch | `executor.py:54` | Guard drifted from intent without detection |
| spec-fidelity inputs: 117.9 KB + 34.2 KB = 152.1 KB (in the dead zone) | File sizes measured | Passes 200 KB guard, exceeds 128 KB kernel limit |
| Same pipeline succeeds with 30 KB spec file | `spec-cross-framework-deep-analysis.md` | Confirms size is the sole variable |
| Test `test_100kb_guard_fallback` mocks subprocess — never validates real CLI | `test_file_passing.py:108` | Zero integration coverage of the fallback path |
| `--file` fallback likely broken (80% confidence) | `arg-too-long-file-fallback-validation.md` | Fallback passes bare paths; `claude --help` documents `file_id:relative_path` format |
| `remediate_executor.py:177` uses `--file` unconditionally | Code review | If `--file` is broken, remediation has never delivered target file content |

### 1.2 Scope Boundary

**In scope**:
- Fix `_EMBED_SIZE_LIMIT` to derive from `MAX_ARG_STRLEN` (Phase 1)
- Change guard to measure full composed `-p` argument (Phase 1)
- Validate `--file` fallback behavior empirically (Phase 0)
- Fix `--file` fallback in all executors if broken (Phase 1.5)
- Per-file size guard and startup warning (Phase 2, optional)

**Out of scope**:
- Stdin prompt delivery (Phase 3 — deferred to v2.26 spike)
- Prompt compression or spec file size reduction strategies
- Windows `CreateProcessW` 32 KB command-line limit (noted, deferred)

## 2. Solution Overview

Replace the miscalibrated magic constant with a derived value that encodes its relationship
to the OS constraint, and change the guard to measure the actual string that becomes the `-p`
argument — not just the embedded portion.

### 2.1 Key Design Decisions

| Decision | Choice | Alternatives Considered | Rationale |
|----------|--------|------------------------|-----------|
| Limit derivation | `_MAX_ARG_STRLEN - _PROMPT_TEMPLATE_OVERHEAD` (~120 KB) | Hardcode 100 KB; query `getrlimit()` at runtime | Self-documenting derivation prevents future drift. `getrlimit` doesn't expose `MAX_ARG_STRLEN` (compile-time constant). |
| Guard measurement | Full composed string (`step.prompt + embedded`) | Embedded-only (current) | Closes template-overhead blind spot identified in L1 debate (88/100 confidence) |
| Template overhead | 8 KB (2.3x largest template at 3.4 KB) | 4 KB (1.14x); dynamic measurement | Conservative margin justified by history of silent drift. Redundant with 1B's full-string check (acts as soft pre-filter). |
| `--file` fallback | Validate empirically before relying on it | Assume working; replace immediately | 80% likely broken per code review; 15-min test resolves ambiguity before committing to fix strategy |

### 2.2 Workflow / Data Flow

```
Before (broken):
  _embed_inputs(files) → len(embedded) <= 200KB? → YES → compose prompt (152 KB)
    → Popen("-p", 152KB_string) → execve() → KERNEL REJECTS (>128 KB) → OSError [Errno 7]

After (fixed):
  _embed_inputs(files) → compose = prompt + embedded → len(composed) <= 120KB?
    → YES: Popen("-p", composed)      → execve() succeeds (< 128 KB)
    → NO:  Popen("--file", paths...)   → execve() succeeds (short args)
```

## 3. Functional Requirements

### FR-1: Derive `_EMBED_SIZE_LIMIT` from `MAX_ARG_STRLEN`

**Description**: Replace `_EMBED_SIZE_LIMIT = 200 * 1024` with a derivation from
`_MAX_ARG_STRLEN = 128 * 1024` minus `_PROMPT_TEMPLATE_OVERHEAD = 8 * 1024`.

**Acceptance Criteria**:
- [ ] `_MAX_ARG_STRLEN`, `_PROMPT_TEMPLATE_OVERHEAD`, and `_EMBED_SIZE_LIMIT` are defined as module-level constants in `executor.py`
- [ ] `_EMBED_SIZE_LIMIT` equals `_MAX_ARG_STRLEN - _PROMPT_TEMPLATE_OVERHEAD` (120 KB)
- [ ] No `import resource` line (dead code from brainstorm draft)
- [ ] Stale `# 100 KB` comment is removed

**Dependencies**: None

### FR-2: Guard full composed `-p` argument

**Description**: Change the embed guard to measure `step.prompt + "\n\n" + embedded`
(the actual value that becomes the `-p` argument), not just `embedded`.

**Acceptance Criteria**:
- [ ] Guard evaluates `len(composed.encode("utf-8")) <= _EMBED_SIZE_LIMIT` where `composed = step.prompt + "\n\n" + embedded`
- [ ] Warning log reports "composed prompt" and actual byte count
- [ ] Fallback fires for inputs that fit individually but exceed limit when combined with prompt

**Dependencies**: FR-1

### FR-3: Rename misleading test

**Description**: Rename `test_100kb_guard_fallback` to `test_embed_size_guard_fallback`
and update docstrings/comments referencing "100KB".

**Acceptance Criteria**:
- [ ] Method renamed to `test_embed_size_guard_fallback`
- [ ] Class docstring updated to reference `_EMBED_SIZE_LIMIT` not "100KB"
- [ ] Test logic unchanged (imports `_EMBED_SIZE_LIMIT`, auto-adapts to new value)

**Dependencies**: FR-1

### FR-4: Test composed-string guard behavior

**Description**: Add a test that verifies the fallback fires when `prompt + embedded`
exceeds the limit even though `embedded` alone fits.

**Acceptance Criteria**:
- [ ] Test creates a file at 90% of `_EMBED_SIZE_LIMIT`
- [ ] Test creates a prompt large enough that combined exceeds limit
- [ ] Asserts file content is NOT in captured prompt (fallback fired)
- [ ] Asserts `--file` flags are present in `extra_args`

**Dependencies**: FR-1, FR-2

### FR-5: Validate `--file` fallback (BLOCKING pre-condition)

**Description**: Empirically test whether `claude --print --file /bare/path` delivers
file content to the model.

**Acceptance Criteria**:
- [ ] Manual test executed: `claude --print -p "What is the secret answer?" --file /tmp/file-test.md`
- [ ] Result documented: WORKING (undocumented behavior) or BROKEN (file content not delivered)
- [ ] If BROKEN: `remediate_executor.py:177` fixed to use inline embedding; fallback path in 3 other executors fixed

**Dependencies**: None (run first)
